Problem

Without groups

  • User -> Resource
  • User -> Resource
  • User -> Resource (chaos)
After

User

  • No direct content permissions
After

Group

  • Role + policy boundary
After

Resource

  • Entries/assets scoped by group rules
Solution

With Access Groups

  • User -> Group -> Resource
  • Clean, repeatable access model
Edge case

User in Group A + Group B

  • Group A + Group B assignments can coexist
  • Effective behavior comes from composed role policies
Architecture rule

Never assign permissions directly to users

  • Assign users to groups
  • Keep policy generation centralized
